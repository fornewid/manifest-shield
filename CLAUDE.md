## AGP Manifest Merger ActionType

BlameLogParser parses AGP's manifest merger blame log to identify the source of each manifest element.

AGP `Actions.ActionType` enum (6 values):

| ActionType | Description | Handled |
|---|---|---|
| `ADDED` | Element was added to the merged manifest | Yes |
| `INJECTED` | Injected by the build system (uses-sdk, package, etc.) | Yes |
| `MERGED` | Merged with another manifest (duplicate permissions, etc.) | Yes |
| `IMPLIED` | Implicitly added because a library targets a lower SDK | Yes |
| `CONVERTED` | Converted to a different element type (nav-graph to intent-filter) | Yes |
| `REJECTED` | Rejected due to conflict (not in the final manifest) | No (excluded) |

Note: `CONVERTED` is used when Navigation's `<nav-graph>` is converted to `<intent-filter>`. Generated by AGP's `NavGraphExpander`.

**Maintenance**: If AGP adds new ActionType values, update the regex patterns in `BlameLogParser`.

## AndroidManifest.xml Elements

When new manifest elements are added, update the following files:

1. `ManifestVisitor.kt` — Add parsing for the new element
2. `ManifestExtraction` — Add new field
3. `models/` — Create new model class
4. `ManifestShieldConfiguration.kt` — Add new boolean flag
5. `ShieldFlags.kt` — Add flag to interface + `applyConfig` extension
6. `ManifestShieldListTask.kt` — Add new category to output
7. `SourcesContentBuilder.kt` — Add new category to source-grouped output
8. `ManifestSourcesDiffTask.kt` — Add to `EnabledCategories`

Reference:
- Manifest elements: https://developer.android.com/guide/topics/manifest/manifest-element
- Application elements: https://developer.android.com/guide/topics/manifest/application-element
